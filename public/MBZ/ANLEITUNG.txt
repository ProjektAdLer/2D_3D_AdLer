╔════════════════════════════════════════════════════════════════╗
║  AdLer MBZ Extractor - Anleitung für Dozenten                 ║
╚════════════════════════════════════════════════════════════════╝

Diese Tools helfen Ihnen, MBZ-Dateien (Moodle Backup) für die
AdLer Engine vorzubereiten.


┌────────────────────────────────────────────────────────────────┐
│ SCHRITT-FÜR-SCHRITT ANLEITUNG                                 │
└────────────────────────────────────────────────────────────────┘

1. MBZ-DATEIEN ABLEGEN
   ► Kopieren Sie Ihre MBZ-Dateien in diesen Ordner (public/MBZ/)
   ► Sie können beliebig viele MBZ-Dateien ablegen

2. ENTPACKEN
   ► Windows:     Doppelklick auf "ENTPACKEN.bat"
   ► macOS/Linux: Doppelklick auf "ENTPACKEN.command"

   Das Script:
   - Scannt automatisch alle MBZ-Dateien in diesem Ordner
   - Entpackt sie nach public/LearningWorlds/
   - Überspringt bereits entpackte Welten
   - Zeigt den Fortschritt an

3. WARTEN
   ► Je nach Größe der MBZ kann das Entpacken 10-60 Sekunden dauern
   ► Bei Problemen werden Warnungen angezeigt
   ► Am Ende erscheint "Vorgang abgeschlossen!"

4. ERGEBNIS PRÜFEN
   ► Entpackte Welten befinden sich in: ../LearningWorlds/
   ► Jede Welt hat einen eigenen Ordner
   ► Die Datei "worlds.json" listet alle verfügbaren Welten auf


┌────────────────────────────────────────────────────────────────┐
│ FÜR LOKALE ENTWICKLUNG                                         │
└────────────────────────────────────────────────────────────────┘

Nach dem Entpacken können Sie die Engine lokal starten:

1. Öffnen Sie ein Terminal/Eingabeaufforderung im Projekt-Root
2. Führen Sie aus: npm start
3. Die Engine öffnet sich im Browser
4. Ihre Welten sind unter "Lernwelten" verfügbar


┌────────────────────────────────────────────────────────────────┐
│ FÜR WEB-DEPLOYMENT (Studenten-Zugriff)                        │
└────────────────────────────────────────────────────────────────┘

Um die Welten für Studenten im Web bereitzustellen:

OPTION A: Kompletter Build
  1. Führen Sie aus: npm run build-showcase
  2. Laden Sie den Ordner "build/" auf Ihren Webserver hoch
  3. Alle entpackten Welten sind automatisch enthalten

OPTION B: Nur neue Welten hochladen (bei bestehendem Deployment)
  1. Laden Sie den Ordner "public/LearningWorlds/" hoch
  2. Überschreiben Sie auf dem Server: /LearningWorlds/
  3. Studenten können die neuen Welten sofort nutzen

OPTION C: GitHub Pages (automatisches Deployment)
  1. Führen Sie aus: npm run build-and-deploy-showcase
  2. Die Welten werden automatisch auf GitHub Pages deployed
  3. URL: https://projektadler.github.io


┌────────────────────────────────────────────────────────────────┐
│ HÄUFIGE FRAGEN                                                 │
└────────────────────────────────────────────────────────────────┘

F: Kann ich mehrere MBZ-Dateien auf einmal entpacken?
A: Ja! Legen Sie alle MBZ-Dateien in diesen Ordner und führen
   Sie das Script einmal aus.

F: Was passiert, wenn ich das Script mehrfach ausführe?
A: Bereits entpackte Welten werden automatisch übersprungen.
   Nur neue MBZ-Dateien werden verarbeitet.

F: Meine MBZ-Datei hat Umlaute im Namen - ist das ein Problem?
A: Manche Umlaute (ä, ö, ü, ß) können beim Entpacken Probleme
   verursachen. Das Script zeigt Warnungen an, aber die meisten
   Elemente funktionieren trotzdem.

F: Wo finde ich die entpackten Welten?
A: In: public/LearningWorlds/
   Jede Welt hat einen eigenen Ordner mit:
   - world.json (Weltdaten)
   - elements/ (Lernelemente: PDFs, H5P, Bilder, etc.)

F: Kann ich entpackte Welten wieder löschen?
A: Ja, löschen Sie einfach den entsprechenden Ordner in
   public/LearningWorlds/ und entfernen Sie den Eintrag aus
   worlds.json
